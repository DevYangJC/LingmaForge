> **文档版本**：V1.0  
> **最后更新**：2026-06-30  
> **文档性质**：施工文档 — 每个文件从哪儿搬到哪儿，精确到行

---

## 一、改造概述

当前所有 74 个 Java 文件混在一个 `com.lingmaforge.backend` 包里，没有模块边界。本次改造**不改 Maven 结构**，仅调整包名和文件位置，按业务域建立清晰的内部分层。

### 改造原则

| # | 规则 | 说明 |
|---|------|------|
| 1 | **不改 pom.xml** | 保持单体 Maven 项目，零构建脚本改动 |
| 2 | **公共层放 common** | 纯数据（Record/Enum/Result），零数据库访问，零业务逻辑 |
| 3 | **基础设施放 infra** | Spring Boot 启动类 + 7 个全局 @Configuration |
| 4 | **业务按域分包** | workbench / auth / creative / billing / doc / admin |
| 5 | **包内自闭** | 每个业务包的 web 只调本包的 service，service 只调本包的 mapper |
| 6 | **预留包只放骨架** | auth/creative/billing/doc/admin 各放一个空 Controller |

### 改造统计

| 类型 | 文件数 |
|------|--------|
| 移动的 Java 源文件 | 74 |
| 移动的资源文件 | 12 |
| 新增的骨架文件 | 5（每个预留包一个空 Controller） |
| 需要修改的文件 | 全部（包名 + import 全局替换） |
| **总计操作** | **91 个文件** |

```java
// 改造前: com.lingmaforge.backend.LingmaForgeBackendApplication.java

    }

```

```java
// 改造后: com.lingmaforge.infra.LingmaForgeApplication.java
package com.lingmaforge.infra;

import org.mybatis.spring.annotation.MapperScan;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.annotation.ComponentScan;

@SpringBootApplication
@ComponentScan(basePackages = "com.lingmaforge")  // 关键：扩大扫描范围到整个 com.lingmaforge
@MapperScan("com.lingmaforge.workbench.mapper")   // 关键：指向 workbench 包的 mapper
public class LingmaForgeApplication {
    public static void main(String[] args) {
        SpringApplication.run(LingmaForgeApplication.class, args);
    }
}
```

**为什么必须加 `@ComponentScan`**：Spring Boot 默认只扫描 `@SpringBootApplication` 所在包及其子包。启动类放到 `com.lingmaforge.infra` 后，默认只扫描 `com.lingmaforge.infra.*`，其他 6 个业务包不会被扫描到，所以必须显式 `basePackages = "com.lingmaforge"`。

## 七、微服务迁移路线（未来参考）

当前改造完成后，每个业务包已经是自闭环的。未来拆分时：

```
步骤 A：给目标包写一个 pom.xml
    workbench 示例：
    <?xml version="1.0" encoding="UTF-8"?>
    <project>
        <parent>
            <groupId>com.lingmaforge</groupId>
            <artifactId>lingmaForge-backend</artifactId>
            <version>0.0.1-SNAPSHOT</version>
        </parent>
        <artifactId>lingma-workbench</artifactId>
    </project>

步骤 B：给目标包写一个独立的 Application.java
    复制 infra 中的最小化配置：数据源、端口、MyBatis-Plus
    启动类只扫描本包：@ComponentScan("com.lingmaforge.workbench")

步骤 C：跨模块 RPC 替换
    把原先的 Java 方法调用改为 REST API 调用
    例如 billing 记录用量：workbench → POST /api/internal/usage/record
```

包结构不需要任何改动，源码文件不需要移动。**只加 pom.xml + 独立 Application + 配置**即可完成微服务拆分。
